About Model Space

Scientific models come in many forms, from equations and simulations to metaphors and diagrams. Instead of defining what counts as a model once and for all, Model Space treats “model” as a multidimensional and prototypical concept. It provides a framework where models can be positioned along various dimensions allowing spatial analysis to complement traditional conceptual analysis.

Model Space thus serves as a meta-model: a flexible environment for exploring how models differ, overlap, and relate across diverse scientific contexts.

Model Space for Fuzzy Categorization is a 3D visualization of a space designed to help fuzzy categorization. It represents models within a 3 dimensional coordinate system defined by:

  • Concreteness (X-axis): How physical or tangible a model is.
  • Mathematicality (Y-axis): The degree of formal mathematical structure.
  • Computationality (Z-axis): How algorithmic or programmable a model is.

Each model appears as a colored sphere whose position corresponds to its values on these three conceptual dimensions. The RGB color of each sphere visually encodes its coordinates in thismodel space.

How to Use

  1. Load Models: Click “Load Models” to upload a JSON file containing model data.
  2. Navigation:
    • Left-click + drag → rotate the view
    • Right-click + drag → pan
    • Scroll → zoom in/out
    • Hover → highlight models

Understanding the Visualization

This conceptual visualization highlights how scientific models vary along three key philosophical dimensions:

Concreteness (X-axis, Red)

From abstract ideas to tangible representations of physical phenomena.

Mathematicality (Y-axis, Green)

From qualitative descriptions to highly formal mathematical systems.

Computationality (Z-axis, Blue)

From conceptual models to executable algorithms and simulations.

Color Coding

The color of each sphere corresponds to its coordinates in the model space:

  • Red → Concreteness
  • Green → Mathematicality
  • Blue → Computationality
